经典理论的现代转世

KV Cache 的工作集与雪崩
1968 年的理论如何解释 LLM 推理

Prefill 节点的 HBM 里塞着许多 session 的 KV cache，谁该留、留多久、超卖了会发生什么——这组问题和 1968 年"给每个进程多少内存"的问题惊人地同构，又在几个关键地方彻底不同。本页沿用前篇的路线：先看清工作集，再走到雪崩边缘。

映射

1. 从分页内存到 KV Cache：一张对照表

只考虑 prefill 节点：一个 session 是一条多轮对话轨迹，每轮到达时追加一段输入；计算这一轮时要对全部历史做 attention，所以该 session 的整个前缀 KV 必须驻留在 HBM 中。对照 1968 年的世界：

三个必须先说清的澄清：

① Miss 不是卡死，是重算。token 文本本身就是“后备存储”——任何 KV 都能从文本重新算出来，只是贵。所以 KV cache 严格说不是请求调页，而是物化视图：数据永远可再生，问题只是你愿不愿意付这个 GPU 账单。真正的三选一是：命中（便宜）、重算（贵，且要挤占别人的空间）、排队（延迟）。

② 部分前缀有部分价值，但只能从尾巴逐出。保留最长连续前缀就能省下等长的重算（vLLM / SGLang 的 prefix cache 正是这么做的）；可要是从中间挖走一块，洞之后的所有 block 实际上都废了。前篇“逐出完整工作集、不要人人剃一刀”的教训，在这里的形态是：逐出永远从 session 的尾部开始。

③ 工作集原理在轮内是被硬件强制的。Denning 要靠调度策略保证“工作集不完整就不许运行”；而 attention 在数学上就要求全前缀同时驻留——原理被直接编译进了算法。于是所有策略自由度都落在轮与轮之间：给谁保留、保留多久、新轮次来了排队还是付费重算。这正是下面几节的全部内容。

概念一的转世

2. 一个 session 的工作集：全有或全无

沿用前篇的路线：先看一个 session 随时间的变化——存活期的经济账（第 3 节）、miss 有多贵（第 4 节），都在单个 session 上就能讲清；多个 session 在 LRU 池里互相挤占，是第 5 节以后的事。

定义（单个 session）　一个 session 在时刻 t、存活期 τ 下的工作集：若它在最近 τ 秒内有过轮次到达，就是它的全部前缀 KV；否则为空集。没有中间状态——前篇的工作集是逐页进出窗口的，这里一旦活跃就是整个前缀，因为 attention 需要完整历史。

局部性也还在，只是换了地方：session 内部没有局部性可言——每轮都要摸全部历史 block，块级访问是“循环全扫”。所有的局部性都在时间维度上：刚说过话的 session 大概率马上还会说话（用户还在对话界面前），沉默越久回来的概率越低。这里的“轮次间隔”（分时系统文献里的老词 think time，指用户读回复、打字的时间，与模型的 thinking 无关）就是前篇引用串局部性的转世。

孤立的 session 永不 miss（首轮除外）——能夺走块的只有别人的压力，而压力落到单个 session 身上只剩一个数字：沉默多久，块会没。这就是存活期 τ；第 5 节会看到池子怎么把它挤出来，这里先当常数。拖动 τ，看图上的标注翻转：

为什么参数是时间，而不是容量？

孤立的 session 里，容量造不出 miss：只要 context 装得下，永远没有东西被逐出。能夺走块的只有别人的压力，而 LRU 按沉默时长索价、不按体积——回来快，90k 也全命中；回来慢，5k 也被吃光。体积只决定被吃时的账单大小（第 4 节的 R）。“超出容量的部分才重算”是池子层的事实——超出多少取决于别人的需求，孤立分析里没有这个量，它在第 5 节登场（那里重算量 = 被挤掉多少补多少）。

本图的简化（缓存理论的“特征时间”近似）：把渐进的尾部侵蚀压扁成“沉默满 τ 瞬间全灭”——短沉默全活、长沉默全灭两端精确，中间“半截被吃”的过渡在第 5 节可见。

留 = 用 HBM 换掉未来的重算；丢 = 用未来的重算换出 HBM。τ 是这笔交换的期限——而它由池子决定，不由 session 自己（第 5 节）。

参数选择

3. 存活期的经济学：两条曲线，一个对偶

既然没人设置 τ，为什么还要画它的曲线？因为 τ 和容量 M 互为对偶——这正是 Denning 学派“变空间 vs 定空间”的经典关系：工作集策略给定 τ、让占用浮动；LRU 池给定 M、把一个等效窗口 τeff 挤出来，恰使平均占用 s(τeff) ≈ M。于是这两条曲线成了容量规划工具：想要命中率 1 − m(τ)，查 m 曲线得到所需存活期 τ*，再查 s(τ*) 读出要为这个 session 付的内存。前篇的恒等式 m(τ) = s′(τ) 转世为：miss 率 = 轮次间隔超过存活期的概率。两条曲线都对第 2 节那个 session 计算：

该 session 的平均 HBM 占用 s(τ)

蓝点与第 2 节的 τ 滑块联动。递增趋饱和：τ 大到一定程度，多留的全是沉默期。

该 session 的回头 miss 率 m(τ)：轮次间隔 > τ 的比例

阶梯每下一级，就是一类停顿被 τ 覆盖住了：常规节奏 8–12 秒先被吃掉，然后是 19 秒，最后是 34 / 36 秒的长停顿。

定量基础

4. Miss 有多贵：99% 命中率是不够的

前篇的支点是"缺页服务时间比一次访存慢 4 个数量级"。这里的对应物不是常数，而是一个随 session 变老不断增大的比值：惩罚比 R = context 长度 ÷ 新一轮长度。5 万 token 上下文的 session 发来 500 token 的一轮：命中时只算 500，全 miss 时要算 50500——R = 100 很日常，长文档场景轻松上千。于是单 session 的计算效率（有效计算占比）是：

e = 1 / (1 + (1 − h) × R)　　其中 h 为命中率，R = context / 轮长

概念二的前奏

5. 把 9 个 session 放进一个 LRU 池

单个 session 的账已经算清，现在把 9 个放进同一块 HBM。池子的规则只有一条（这就是 vLLM / SGLang 的真实行为）：块进来就留下，没有过期时间；只有当新的分配需要空间时，才从“最久没说话”的 session 的尾部开始逐出。驱逐的因是容量，不是时钟——其余一切都是这条规则的后果。读图前先记住四件事：

① 上图的一个点 = 一轮输入的到达，不是内存块。蓝点 = 命中：它的历史块全在池中，这一轮只算新增的一小截；红点 = 被挤过：沉默期间历史被别人挤掉了一部分或全部，这一轮要先重算缺失的前缀（第 4 节的 R 倍账单）。

② 没有窗口了。块不会自己过期，只会被新需求挤出去。高亮的行 = 此刻在池中还有块的 session——包括已经死了但还没被挤掉的。

③ 下图的堆叠 = 各 session 实际驻留的块，总和永远不超过容量 M。虚线阶梯是“如果内存无限、本可以全部缓存”的总量；虚线与堆顶之间的浅红楔子 = 已被挤出的块——谁回来谁付重算。挤占就发生在带子被削顶的那些瞬间：别人一说话，最老 session 的尾巴被当场吃掉。

④ 等效窗口 τeff 是被挤出来的读数：池中最老的完整幸存者闲置了多久。M 越小、说话的人越多，τeff 被压得越短；当它短于典型轮次间隔（本例 8–12 秒），几乎每个回头轮都变红——那就是雪崩的定义（第 6 节）。

轮次到达（每行一个 session；蓝点 = 命中，红点 = 历史被挤过、需重算）

红蓝是整段模拟的结果，只随 M 变化：M 拉大，红点变少；M 拉小，挤占提前、红点蔓延。

LRU 池：各 session 实际驻留的块（堆叠，总和 ≤ M；单位：千 token）

虚线阶梯 = 若内存无限本可缓存的总量；虚线与堆顶之间的浅红区 = 已被挤出的块。注意死掉的 session（会话 2、9）的带子也会一直赖到被挤才走。

t = 　池中 session 　已用 / 容量 　已挤出 　等效窗口 τeff

值得盯着看的三个现象：

① 死人不会自己走。会话 2 在 46 秒后永远沉默（用户关了网页），但它的带子一直赖在池里——LRU 不清理死人，只是在需要空间时优先吃掉最久没说话的。死亡不可观测，而 LRU 赌的恰恰是“最老的最可能已经死了”：逐出死人零代价，这是它赌对的部分；误伤长停顿的活人（会话 3），是它赌输的部分。

② 挤占有清晰的因果链。盯住会话 3：它停顿的 61 秒里，如果池子饱和过，它的带子会被后来者一口口削掉；等它回来，那个点就是红的——别人说话 → 它被逐 → 它回来 → 它付账。把 M 拉大到需求线之上，同一个点变蓝：miss 从来不是它自己的错，是容量的错。

③ 压力自己会涨。带子只增不减（context 单调增长），所以 session 数量不变、节奏不变，虚线需求也在爬坡——运行得越久，τeff 被压得越短。负载恒定的系统也会自己滑向雪崩，这是和前篇最隐蔽的差别。

概念二的转世

6. 雪崩线：HBM、session 数与 context 的三角关系

第 5 节的池子只有 9 个 session，τeff 只是被压短；现在把负载推到极限，看它被压穿的瞬间。设 N 个 session 平均 context 为 c̄、每轮新增 δ = 0.5k token。总需求 N·c̄ ≤ M 时人人全命中；一旦超过，LRU 池会遭遇缓存理论里的经典病理——循环扫描略大于缓存的集合，命中率不是缓降而是阶跃归零（每个 session 每轮全扫自己的前缀，回来时自己的块恰好是最旧的，刚好被挤光；数据库 buffer pool 的 scan thrashing 同款）。此时 τeff 恰好被压到典型轮次间隔以下——第 5 节的读数和这里的悬崖是同一件事的两种说法。前篇的悬崖需要一个教学用的幂指数，这里的悬崖是结构性的，位置就在：

雪崩线：M = N × c̄　——HBM 越大养得起的 session 越多（线的斜率就是 c̄），跨过线即雪崩。

两条线在跨线后都会下坠（超额 session 的重算是真实工作量，谁也变不走），但读法完全不同：LRU 下所有 session 一起沉没；准入控制下驻留的 K 个 session 各自仍是 100% 效率，损失被隔离在超额者身上——M=400、c̄=50 时，N=9 的总 goodput 是 8.3% vs LRU 的 1%，差 8 倍，且随 N 增大持续领先（见数据表）。

模型公式与推导

块级 LRU：session 回来时，缓存里是比它更新的 N−1 个 session 的 context，留给它的空间是 M − (N−1)·c̄。命中率 h = clamp((M − (N−1)·c̄) / c̄, 0, 1)——在宽度仅 c̄ 的过渡带内从 1 跌到 0。每轮实际算量 = δ + (1−h)·c̄，goodput = δ / (δ + (1−h)·c̄)。

session 级准入：K = ⌊M / c̄⌋ 个 session 驻留、全命中；其余 N−K 个不驻留，每轮自付全量重算，但不侵占驻留者的空间。goodput = N·δ / (K·δ + (N−K)·(δ + c̄))。

注意与前篇的不同：OS 里被挂起的进程不消耗 CPU，绿线是平的；这里超额 session 开口说话就得算，绿线也会下垂——准入控制不能消灭工作量，它消灭的是互相踩踏：驻留的 K 个 session 效率恒为 100%，损失被隔离在超额者自己身上。若在门口直接排队/限流（不处理超额轮次），goodput 可保持 ~100%，代价是排队延迟——那才是前篇"整体挂起"的严格对应物。

比前篇更险恶的一点：雪崩发生时 GPU 利用率是 100%。OS 抖动的体征是 CPU 空转、磁盘狂响，一眼可诊断；KV 雪崩时监控面板一片健康——GPU 在满负荷地重算历史。必须换指标才能看见它：goodput、cache 命中率、TTFT，缺一不可。

正反馈不需要任何人犯错，有两条自动回路。Denning 的螺旋靠调度器误判（“CPU 空闲 → 多放进程”）；这里的版本是全自动的：① Little 定律堆积——每轮变慢 → 在途请求滞留 → 同时驻留的 session 变多 → 池子更挤 → 更慢；② 重试风暴——TTFT 超时 → 客户端重发 → 同一个长 prefill 提交两遍 → 需求进一步过载。1968 年那个火上浇油的调度器，今天叫 Little 定律加超时重试。

药方

7. 药方：还是工作集原理，但形态变了

前篇的药方是一句话：Σ|Wi| ≤ M，装不下就整体挂起。转世到 KV cache，它裂解成一组互相配合的机制：

把 τeff 当读数，把 M 和准入当旋钮

单机池子里没人设置 τ——LRU 把它挤成 τeff。真正的决策是容量规划（第 3 节的对偶曲线：让典型 session 的存活期越过膝点）和下面的准入控制；显式 TTL 只出现在多级缓存与外部 KV 存储的配额上。

Session 粒度逐出，尾部优先

要让位就整条带子抽走（或从尾部截断），绝不从中间挖洞、绝不让新 session 零敲碎打地踩别人的前缀——那是雪崩的引信。

门口准入，而非池内互踩

需求过线时，宁可让新轮次排队/限流（付延迟），也不放进来触发全池重算（付吞吐）。vLLM 的调度器不见 block 不排批、满池时整序列 preempt（recompute 或 swap 二选一——正是"重算 vs 换页"），都是这个原则。

亲和路由：别弄丢工作集

多个 prefill 节点时，把 session 路由回持有其缓存的节点（对应 OS 的进程迁移代价）。SGLang 的 radix tree 前缀共享、Mooncake 的 KVCache 中心调度，都是把"工作集在哪"当成一等公民的调度。

还有一个前篇没有的对手值得点名：共享前缀。公共 system prompt 是所有 session 工作集的交集——1968 年共享代码段的转世，也是现代 prefix cache 命中率的最大来源。它不改变雪崩的数学，但改变了 c̄ 的账。

收尾

8. 像与不像：六个本质差异

① 局部性的单位变了

页 → session 前缀。session 内部每轮全扫历史，没有块级局部性；所有局部性都在"谁最近说过话"这个维度上。

② 工作集只增不减，死亡不可观测

进程工作集随阶段漂移；session 前缀单调生长直到无声无息地死去。逐出只能靠赌：LRU 赌最久沉默的最可能已死——赌对免费，赌错付 R 倍。

③ 后备存储是"重算"

缺页读磁盘，miss 付 GPU：数据永远可再生，代价 ∝ 丢失量。KV cache 是物化视图，不是请求调页。

④ 惩罚比随年龄增长

T/访存 ≈ 10⁴ 是物理常数；R = context/轮长逐轮变大。负载不变，系统也会滑向悬崖。

⑤ 雪崩是隐形的

OS 抖动 CPU 空转，可见；KV 雪崩 GPU 100% 忙于无用功。诊断指标必须换成 goodput / 命中率 / TTFT。

⑥ 悬崖更陡，且无需拟合
